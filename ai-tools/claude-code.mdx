Claude Code — официальный CLI-инструмент Anthropic. Это руководство поможет вам настроить Claude Code для помощи в написании и поддержке документации.

## Требования

- Активная подписка Claude (Pro, Max или доступ к API)

## Установка

1. Установите Claude Code глобально:

  ```bash
  npm install -g @anthropic-ai/claude-code
```

2. Перейдите в папку с документацией.
3. (Опционально) Добавьте файл `CLAUDE.md` ниже в ваш проект.
4. Запустите `claude` для старта.

## Создайте `CLAUDE.md`

Создайте файл `CLAUDE.md` в корне репозитория документации, чтобы обучить Claude Code вашим стандартам документации:

````markdown
# Документация Mintlify

## Рабочие отношения
- Вы можете возражать идеям — это может привести к лучшей документации. Ссылайтесь на источники и объясняйте свои рассуждения
- ВСЕГДА спрашивайте уточнения вместо того, чтобы делать предположения
- НИКОГДА не лгите, не угадывайте и не выдумывайте информацию

## Контекст проекта
- Формат: MDX файлы с YAML frontmatter
- Конфигурация: docs.json для навигации, темы, настроек
- Компоненты: компоненты Mintlify

## Стратегия контента
- Документируйте ровно столько, сколько нужно для успеха пользователя — не слишком много, не слишком мало
- Приоритет на точность и удобство использования информации
- Делайте контент вечнозелёным, когда возможно
- Ищите существующую информацию перед добавлением новой. Избегайте дублирования, если это не делается стратегически
- Проверяйте существующие паттерны для консистентности
- Начинайте с минимально разумных изменений

## Требования frontmatter для страниц
- title: Чёткий, описательный заголовок страницы
- description: Краткое описание для SEO/навигации

## Стандарты написания
- Второе лицо ("вы")
- Предварительные требования в начале процедурного контента
- Тестируйте все примеры кода перед публикацией
- Соответствуйте стилю и форматированию существующих страниц
- Включайте базовые и продвинутые сценарии использования
- Теги языка на всех блоках кода
- Alt-текст на всех изображениях
- Относительные пути для внутренних ссылок

## Рабочий процесс Git
- НИКОГДА не используйте --no-verify при коммите
- Спрашивайте как обработать незакоммиченные изменения перед началом
- Создавайте новую ветку, когда нет чёткой ветки для изменений
- Коммитьте часто в процессе разработки
- НИКОГДА не пропускайте и не отключайте pre-commit хуки

## Не делайте
- Пропускать frontmatter на любом MDX файле
- Использовать абсолютные URL для внутренних ссылок
- Включать непротестированные примеры кода
- Делать предположения — всегда спрашивайте уточнения
````
