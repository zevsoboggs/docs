Настройте AI-ассистент Cascade в Windsurf для помощи в написании и поддержке документации. Это руководство показывает как настроить Windsurf специально для вашего рабочего процесса с документацией Mintlify.

## Требования

- Установленный редактор Windsurf
- Доступ к репозиторию документации

## Правила рабочего пространства

Создайте правила рабочего пространства, которые предоставят Windsurf контекст о вашем проекте документации и стандартах.

Создайте `.windsurf/rules.md` в корне проекта:

````markdown
# Правило технического писательства Mintlify

## Контекст проекта

- Это проект документации на платформе Mintlify
- Мы используем MDX файлы с YAML frontmatter
- Навигация настраивается в `docs.json`
- Мы следуем лучшим практикам технического писательства

## Стандарты написания

- Используйте второе лицо ("вы") для инструкций
- Пишите в активном залоге и настоящем времени
- Начинайте процедуры с предварительных требований
- Включайте ожидаемые результаты для основных шагов
- Используйте описательные заголовки с ключевыми словами
- Пишите краткие, но информативные предложения

## Обязательная структура страницы

Каждая страница должна начинаться с frontmatter:

```yaml
---
title: "Чёткий, конкретный заголовок"
description: "Краткое описание для SEO и навигации"
---
```

## Компоненты Mintlify

### Выноски

- `<Note>` для полезной дополнительной информации
- `<Warning>` для важных предупреждений и критических изменений
- `<Tip>` для лучших практик и экспертных советов
- `<Info>` для нейтральной контекстной информации
- `<Check>` для подтверждений успеха

### Примеры кода

- При необходимости включайте полные, работающие примеры
- Используйте `<CodeGroup>` для примеров на нескольких языках
- Указывайте теги языка на всех блоках кода
- Используйте реалистичные данные, не заполнители
- Используйте `<RequestExample>` и `<ResponseExample>` для документации API

### Процедуры

- Используйте компонент `<Steps>` для последовательных инструкций
- Включайте шаги проверки с компонентами `<Check>` когда уместно
- Разбивайте сложные процедуры на меньшие шаги

### Организация контента

- Используйте `<Tabs>` для контента, специфичного для платформы
- Используйте `<Accordion>` для прогрессивного раскрытия
- Используйте `<Card>` и `<CardGroup>` для выделения контента
- Оборачивайте изображения в компоненты `<Frame>` с описательным alt-текстом

## Требования к документации API

- Документируйте все параметры с `<ParamField>`
- Показывайте структуру ответа с `<ResponseField>`
- Включайте примеры успеха и ошибок
- Используйте `<Expandable>` для вложенных свойств объектов
- Всегда включайте примеры аутентификации

## Стандарты качества

- Тестируйте все примеры кода перед публикацией
- Используйте относительные пути для внутренних ссылок
- Включайте alt-текст для всех изображений
- Обеспечьте правильную иерархию заголовков (начинайте с h2)
- Проверяйте существующие паттерны для консистентности
````
