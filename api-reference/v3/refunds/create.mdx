---
title: 'Создать возврат'
description: 'POST /api/v3/invoices/{id}/refund — возврат (идемпотентно)'
---

Создаёт заявку на возврат по оплаченному счёту (полный или частичный). Заявку подтверждает
администратор Love&Pay; результат приходит вебхуком `refund.*`. Требует scope
**`refunds:write`**, поддерживает `idempotency-key`.

**Endpoint:** `POST https://loveandpay.io/api/v3/invoices/{id}/refund`

Заголовки — см. [Аутентификация v3](/api-reference/v3/authentication). `{id}` — UUID или номер счёта.

## Тело запроса

<ParamField body="amount" type="number">
  Сумма возврата в рублях. **Необязательно** — без неё возвращается весь остаток. Частичный — можно несколько раз до исчерпания остатка.
</ParamField>

<ParamField body="reason" type="string">
  Причина возврата (до 1000 символов).
</ParamField>

<RequestExample>
```javascript Node.js

```

## Статусы

| Статус | Значение |
|---|---|
| `PENDING` | ждёт решения администратора |
| `APPROVED` | утверждён, исполняется |
| `COMPLETED` | возврат выполнен ✅ |
| `FAILED` | не выполнен (см. `executionError`) |
| `REJECTED` | отклонён |

Отслеживайте результат по вебхукам `refund.completed` / `refund.failed` / `refund.rejected`
(см. [Вебхуки](/guides/webhooks)) или опросом [GET /api/v3/refunds/{id}](/api-reference/v3/refunds/get).
