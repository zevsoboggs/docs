---

API v3 — флагманская версия с упором на **безопасность, скорость и стабильность**. В отличие
от v2 (общий HMAC-секрет), доверие строится на **клиентском сертификате**: вы подписываете
запросы своим приватным ключом, который **никогда не покидает ваш сервер** и не хранится у нас.

## Что нового в v3

<CardGroup cols={2}>
  <Card title="Сертификатное доверие" icon="certificate">
    Асимметричная подпись (ECDSA P-256). Приватный ключ клиента у нас не хранится — компрометация нашей БД не даёт подделать запрос.
  </Card>
  <Card title="Идемпотентность" icon="fingerprint">
    `Idempotency-Key` — повторный запрос возвращает тот же результат, без дубль-платежа.
  </Card>
  <Card title="Защита от повторов" icon="shield-halved">
    Уникальный `nonce` на каждый запрос — перехваченный запрос нельзя воспроизвести.
  </Card>
  <Card title="Права сертификата (scopes)" icon="key">
    Least privilege: сертификат можно ограничить (`invoices:write`, `refunds:read`…). Утёкший серт не даёт всё.
  </Card>
  <Card title="Шифрование полей" icon="lock">
    Опциональное E2E-шифрование тела и ответа (ECDH P-256 + AES-256-GCM).
  </Card>
  <Card title="Чистый контракт" icon="code">
    Единый envelope, cursor-пагинация, машиночитаемые ошибки с флагом `retryable`.
  </Card>
</CardGroup>

## Базовый URL

```
https://loveandpay.io/api/v3
```

## Ключевые отличия от v2

| Функция | API v2 | API v3 |
|---|---|---|
| Аутентификация | HMAC-SHA256 (общий секрет) | Клиентский сертификат (асимметричная подпись) |
| Приватный ключ | секрет у обеих сторон | **только у клиента**, у нас не хранится |
| Защита от повтора | окно времени (5 мин) | окно времени **+ nonce** |
| Идемпотентность | — | `Idempotency-Key` |
| Права ключа | всё или ничего | **scopes** (least privilege) |
| Шифрование полей | — | опционально (ECDH + AES-GCM) |
| Пагинация | offset | cursor |
| Формат ответа | `{ success, ... }` | envelope `{ ok, data, error, requestId }` |

## Формат ответа

Все ответы v3 — единый envelope:

```json
{
  "ok": true,
  "data": { },
  "requestId": "5f3c…"
}
```

Ошибка — машиночитаемая, с флагом повторяемости:

```json
{
  "ok": false,
  "error": {
    "code": "IDEMPOTENCY_CONFLICT",
    "message": "Idempotency-Key reused with a different request or still processing",
    "retryable": true,
    "docsUrl": "https://docs.loveandpay.io/api-reference/v3/authentication"
  },
  "requestId": "5f3c…"
}
```

<Info>
`retryable: true` означает, что запрос можно безопасно повторить (временная проблема,
конфликт идемпотентности, сбой провайдера). `retryable: false` — повтор без изменений
не поможет.
</Info>

## С чего начать

<Steps>
  <Step title="Выпустите сертификат">
    В кабинете: [Настройки → API v3 · Сертификаты](https://loveandpay.io/dashboard/settings?tab=v3-certs). Скачайте `.crt` и `.key` (приватный ключ показывается **один раз**).
  </Step>
  <Step title="Настройте подпись запросов">
    См. [Аутентификация](/api-reference/v3/authentication) — заголовки, подпись, nonce.
  </Step>
  <Step title="Создайте первый счёт">
    [POST /v3/invoices](/api-reference/v3/invoices/create).
  </Step>
</Steps>

## Совместимость

API v2 остаётся полностью поддерживаемым. Переход на v3 — добровольный. Вебхуки — общие
для обеих версий (см. [Вебхуки](/guides/webhooks)).
