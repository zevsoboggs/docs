---
title: 'Создать счёт'
description: 'POST /api/v3/invoices — создание счёта с QR (идемпотентно)'
---

Создаёт счёт и регистрирует QR (СБП). Требует scope **`invoices:write`**. Поддерживает
идемпотентность через `idempotency-key`.

**Endpoint:** `POST https://loveandpay.io/api/v3/invoices`

Заголовки аутентификации — см. [Аутентификация v3](/api-reference/v3/authentication).

## Тело запроса

<ParamField body="amount" type="number" required>
  Сумма в рублях (например `1500.50`).
</ParamField>

<ParamField body="description" type="string">
  Назначение платежа.
</ParamField>

<ParamField body="customer" type="object">
  Данные клиента: `email`, `name`, `phone` (все необязательны).
</ParamField>

<ParamField body="expiresInHours" type="number" default="1">
  Срок жизни счёта в часах.
</ParamField>

<ParamField body="paymentMethod" type="string">
  Метод оплаты для провайдеров, где применимо (напр. `fps`).
</ParamField>

<ParamField body="successUrl" type="string">
  URL возврата клиента после оплаты.
</ParamField>

<ParamField body="metadata" type="object">
  Произвольные данные — вернутся в ответе и вебхуках (`metadata`).
</ParamField>

<RequestExample>
```javascript Node.js
// callV3 — см. страницу «Аутентификация v3»
const out = await callV3('POST', '/api/v3/invoices', {
  amount: 1500.50,
  description: 'Заказ #123',
  customer: { email: 'client@example.com' },
  metadata: { orderId: '123' },
}, { idempotencyKey: crypto.randomUUID() })

console.log(out.data.invoice.qr.payload)
```
</RequestExample>

<Note>
При оплате счёта прилетит вебхук `invoice.paid`, при создании — `invoice.created`.
См. [Вебхуки](/guides/webhooks).
</Note>
