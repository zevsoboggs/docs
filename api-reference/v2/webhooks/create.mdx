<Warning>
**Создание вебхуков через публичный API отключено.**

Это сделано осознанно для безопасности — чтобы скомпрометированный API-ключ не позволил атакующему перенаправить ваши финансовые уведомления на свой сервер.
</Warning>

## Как создать вебхук

Только через веб-кабинет:

<Steps>
  <Step title="Откройте раздел">
    [Панель управления → Вебхуки](https://loveandpay.io/dashboard/webhooks)
  </Step>
  <Step title="Нажмите «Создать вебхук»">
    Заполните **URL** (HTTPS), отметьте нужные **события**, при желании — **описание**.
  </Step>
  <Step title="Сохраните Webhook Secret">
    После создания будет показан секрет для проверки HMAC-подписи входящих сообщений.
  </Step>
  <Step title="Протестируйте">
    Кнопка **«Тест»** в карточке вебхука отправит примерный POST на ваш URL.
  </Step>
</Steps>

Полное руководство — [Вебхуки](/guides/webhooks).

## Доступные через API операции

Через API доступны только **read-only**-операции:

- [GET /api/v2/webhooks](/api-reference/v2/webhooks/list) — список ваших вебхуков
- [GET /api/v2/webhooks/{id}/events](/api-reference/v2/webhooks/events) — история доставки

Для изменения/удаления — также используйте веб-кабинет.

## Почему нет POST через API?

**Защита от компрометации**:
- Если ваш API-ключ утечёт, злоумышленник не сможет перенаправить вебхуки на свой сервер
- Управление точкой получения денежных уведомлений требует второго фактора защиты — авторизации в кабинете

Это стандартная индустриальная практика — у Stripe, PayPal и большинства провайдеров создание вебхуков через API также требует дополнительной авторизации.
