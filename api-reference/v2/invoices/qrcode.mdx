## Как устроен QR

<Warning>
**СБП QR-код живёт ограниченное время — как правило, около 5 минут.**

Сам счёт действует **1 час**. Пока счёт в статусе `PENDING`, QR можно
переполучать сколько угодно — система **автоматически пересоздаёт** его,
когда срок истёк. Ориентируйтесь на поля `remainingSeconds` и
`qrValiditySeconds` из ответа, а не на фиксированное число.
</Warning>

Этот эндпоинт всегда возвращает **актуальный** QR:
- если текущий QR ещё действует — отдаёт его и `remainingSeconds` до истечения;
- если истёк (или передан `?refresh=1`) — пересоздаёт новый у провайдера.

`id` может быть как **UUID** счёта, так и его **номером** (`INV-...`).

## Рекомендуемый сценарий отображения

<Steps>
  <Step title="Создайте счёт">
    `POST /api/v2/invoices` — в ответе уже есть первый `qrCode` и `qrPayload`.
  </Step>
  <Step title="Показывайте QR клиенту">
    Рисуйте `qrCode` (картинка) или `payload` (ссылка `qr.nspk.ru/...`).
  </Step>
  <Step title="Обновляйте до истечения">
    Каждые **&lt; 5 минут** (например, раз в 60–120 сек) запрашивайте
    `GET /invoices/{id}/qrcode` и перерисовывайте QR. Ориентируйтесь на
    `remainingSeconds`.
  </Step>
  <Step title="Останавливайтесь по статусу">
    Когда `status` стал `PAID` / `EXPIRED` / `CANCELLED` — прекратите обновление.
  </Step>
</Steps>

## Заголовки

<ParamField header="x-api-key" type="string" required>Ваш API ключ</ParamField>
<ParamField header="x-timestamp" type="string" required>Unix timestamp в миллисекундах</ParamField>
<ParamField header="x-signature" type="string" required>HMAC-SHA256 подпись</ParamField>

## Path параметры

<ParamField path="id" type="string" required>
  UUID счёта **или** номер счёта (`INV-...`)
</ParamField>

## Query параметры

<ParamField query="refresh" type="string">
  `1` или `true` — принудительно пересоздать QR, даже если текущий ещё действует.
  (То же делает `POST` на этот URL.)
</ParamField>

## Ответ

<ResponseField name="qrCode" type="string">Картинка QR в формате data URL (`data:image/png;base64,...`)</ResponseField>
<ResponseField name="payload" type="string">СБП-payload — ссылка `https://qr.nspk.ru/...`</ResponseField>
<ResponseField name="invoiceNumber" type="string">Номер счёта</ResponseField>
<ResponseField name="status" type="string">Статус: `PENDING` / `PAID` / `EXPIRED` / `CANCELLED`</ResponseField>
<ResponseField name="regenerated" type="boolean">`true`, если QR пересоздан в этом запросе</ResponseField>
<ResponseField name="remainingSeconds" type="number">Сколько секунд текущий QR ещё действителен</ResponseField>
<ResponseField name="qrValiditySeconds" type="number">Полный срок жизни QR провайдера (300 для СБП)</ResponseField>
<ResponseField name="qrExpiresAt" type="string">Время истечения текущего QR (ISO 8601)</ResponseField>
<ResponseField name="qrCodeCreatedAt" type="string">Время создания текущего QR (ISO 8601)</ResponseField>

<RequestExample>
```bash cURL
curl -X GET "https://loveandpay.io/api/v2/invoices/INV-260520-0042/qrcode" \
  -H "x-api-key: pk_live_xxx" \
  -H "x-timestamp: 1716207600000" \
  -H "x-signature: a1b2c3d4e5f6..."
```

```javascript JavaScript
const crypto = require('crypto');
const API_KEY = 'pk_live_xxx';
const SECRET_KEY = 'sk_live_xxx';

async function getFreshQr(invoiceId) {
  const timestamp = Date.now().toString();
  const method = 'GET';
  const path = `/api/v2/invoices/${invoiceId}/qrcode`; // БЕЗ query в подписи!
  const bodyHash = crypto.createHash('sha256').update('').digest('hex');
  const message = `${method}${path}${timestamp}${bodyHash}`;
  const signature = crypto.createHmac('sha256', SECRET_KEY).update(message).digest('hex');

  const res = await fetch('https://loveandpay.io' + path, {
    headers: { 'x-api-key': API_KEY, 'x-timestamp': timestamp, 'x-signature': signature },
  });
  return res.json();
}

// Поллинг: обновляем QR пока счёт PENDING
async function showInvoiceQr(invoiceId, imgEl) {
  while (true) {
    const d = await getFreshQr(invoiceId);
    if (['PAID', 'EXPIRED', 'CANCELLED'].includes(d.status)) break;
    imgEl.src = d.qrCode;
    const wait = Math.max(15, (d.remainingSeconds || 60) - 20) * 1000; // обновить заранее
    await new Promise(r => setTimeout(r, wait));
  }
}
```
</RequestExample>

<Note>
В HMAC-подпись включается **PATH без query** (`/api/v2/invoices/{id}/qrcode`),
а `?refresh=1` добавляется только к URL запроса.
</Note>
