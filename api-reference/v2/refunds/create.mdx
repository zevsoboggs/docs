---
title: 'Создать возврат'
description: 'Заявка на возврат по оплаченному счёту (полный или частичный)'
api: 'POST https://loveandpay.io/api/v2/invoices/{id}/refund'
---

Создаёт **заявку на возврат** по оплаченному счёту. Заявка проходит модерацию: её
подтверждает администратор Love&Pay, после чего возврат исполняется у платёжного
провайдера. О результате вы узнаёте по [вебхукам](/guides/webhooks) `refund.completed`
или `refund.failed`.

<Info>
Возврат возможен только для счетов в статусе **PAID** и по провайдерам, которые его
поддерживают (СБП). Можно вернуть **всю сумму** или **часть** — и даже несколькими
частями, пока не исчерпан остаток.
</Info>

## Заголовки

<ParamField header="x-api-key" type="string" required>
  Ваш API ключ
</ParamField>

<ParamField header="x-timestamp" type="string" required>
  Unix timestamp в миллисекундах
</ParamField>

<ParamField header="x-signature" type="string" required>
  HMAC-SHA256 подпись
</ParamField>

## Path параметры

<ParamField path="id" type="string" required>
  UUID счёта или его номер (`INV-...`)
</ParamField>

## Тело запроса

<ParamField body="amount" type="number">
  Сумма возврата в рублях. **Необязательно** — если не указать, вернётся **весь
  остаток** по счёту. Для частичного возврата укажите сумму от `0.01` до текущего
  остатка.
</ParamField>

<ParamField body="reason" type="string">
  Причина возврата (до 1000 символов). Отображается администратору при модерации.
</ParamField>

## Статусы заявки

| Статус | Значение |
|---|---|
| `PENDING` | Заявка создана, ожидает решения администратора |
| `APPROVED` | Утверждена, возврат исполняется у провайдера |
| `COMPLETED` | Деньги возвращены клиенту ✅ |
| `FAILED` | Возврат не выполнен (см. `executionError`) |
| `REJECTED` | Администратор отклонил заявку |

```javascript JavaScript
const crypto = require('crypto');

const API_KEY = 'pk_live_xxx';
const SECRET_KEY = 'sk_live_xxx';

const invoiceId = 'INV-260716-0042';
const timestamp = Date.now().toString();
const method = 'POST';
const path = `/api/v2/invoices/${invoiceId}/refund`;
const body = JSON.stringify({ amount: 500.00, reason: 'Клиент отменил заказ' });

const bodyHash = crypto.createHash('sha256').update(body).digest('hex');
const message = `${method}${path}${timestamp}${bodyHash}`;
const signature = crypto.createHmac('sha256', SECRET_KEY).update(message).digest('hex');

const response = await fetch('https://loveandpay.io' + path, {
  method,
  headers: {
    'x-api-key': API_KEY,
    'x-timestamp': timestamp,
    'x-signature': signature,
    'Content-Type': 'application/json'
  },
  body
});

console.log(await response.json());
```

```python Python
import hashlib
import hmac
import json
import time
import requests

API_KEY = 'pk_live_xxx'
SECRET_KEY = 'sk_live_xxx'

invoice_id = 'INV-260716-0042'
timestamp = str(int(time.time() * 1000))
method = 'POST'
path = f'/api/v2/invoices/{invoice_id}/refund'
body = json.dumps({'amount': 500.00, 'reason': 'Клиент отменил заказ'})

body_hash = hashlib.sha256(body.encode()).hexdigest()
message = f"{method}{path}{timestamp}{body_hash}"
signature = hmac.new(SECRET_KEY.encode(), message.encode(), hashlib.sha256).hexdigest()

response = requests.post(
    'https://loveandpay.io' + path,
    headers={
        'x-api-key': API_KEY,
        'x-timestamp': timestamp,
        'x-signature': signature,
        'Content-Type': 'application/json'
    },
    data=body
)

print(response.json())
```
