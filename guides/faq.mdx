## Платежи и счета

<AccordionGroup>
  <Accordion title="Сколько действует QR-код для оплаты?">
    QR-код от банка по правилам НСПК живёт **5 минут**. Но **внутренний счёт Love&Pay активен 1 час** — внутри этого времени мы **автоматически пересоздаём QR каждые 5 минут**. Клиент всегда видит свежий код, ничего делать не нужно. Подробности — [Жизненный цикл счёта](/guides/invoice-lifecycle).
  </Accordion>

  <Accordion title="Какая минимальная сумма счёта?">
    **10 ₽** (1000 копеек). Это техническое ограничение системы.
  </Accordion>

  <Accordion title="Какая максимальная сумма?">
    Зависит от вашего терминала / провайдера и от лимитов СБП на стороне банка клиента. Стандартный лимит СБП на одну операцию — 1 000 000 ₽ в сутки на физлицо.
  </Accordion>

  <Accordion title="Как быстро приходят деньги?">
    Через СБП — **мгновенно**, поступление на ваш счёт обычно в течение секунд. Зависит от вашего банка-эквайера.
  </Accordion>

  <Accordion title="Можно ли создать счёт в другой валюте?">
    Да, через [POS-терминал](https://loveandpay.io/dashboard/pos). Выбираете валюту клиента (USD, EUR, TRY, THB и т.д.), вводите сумму — система автоматически конвертирует в рубли по актуальному курсу с вашей наценкой. Клиент оплачивает рублями через СБП.
  </Accordion>

  <Accordion title="Что произойдёт если клиент не оплатит за час?">
    Счёт автоматически перейдёт в статус `EXPIRED`. Сработает webhook `invoice.expired` (если настроен). Деньги клиенту не списываются — он просто не успел оплатить. Создайте новый счёт через UI или API.
  </Accordion>
</AccordionGroup>

## API и ключи

<AccordionGroup>
  <Accordion title="Где взять API-ключи?">
    [Профиль → API](https://loveandpay.io/dashboard/profile).

    - **API Key** (постоянный) — UUID-ключ, видим всегда.
    - **Secret Key** (для API v2 с HMAC) — показывается **один раз** при генерации. Не сохранили — придётся сгенерировать новый.
  </Accordion>

  <Accordion title="API v1 или v2 — что выбрать?">
    Для нового интегрирования — **только v2** (HMAC-SHA256 подпись, защита от replay-атак, расширенный набор endpoints).
    v1 — устаревший, поддерживается для совместимости со старыми клиентами.
  </Accordion>

  <Accordion title="Что делать если потерял Secret Key?">
    Откройте [Профиль → API → API v2](https://loveandpay.io/dashboard/profile) и нажмите **«Сгенерировать новый секрет»**. Старый перестаёт работать **сразу**, без grace-периода. Обновите его на своей стороне.
  </Accordion>

  <Accordion title="Какие rate limits?">
    По умолчанию **100 запросов/минуту** и **1000/час** на одного партнёра. При превышении — HTTP 429. Текущие лимиты вашего аккаунта видны в [Профиль → API](https://loveandpay.io/dashboard/profile).
  </Accordion>

  <Accordion title="Получаю TIMESTAMP_EXPIRED — почему?">
    Время на вашем сервере рассинхронизировано с UTC более чем на 5 минут. Настройте NTP-синхронизацию (`timedatectl set-ntp true` на Linux).
  </Accordion>

  <Accordion title="Получаю INVALID_SIGNATURE — почему?">
    Самые частые причины:
    - В подпись попали query-параметры (нужен **только PATH**: `/api/v2/invoices`, без `?limit=10`)
    - Тело запроса в подписи отличается от того что отправили (порядок ключей, пробелы)
    - Используется неверный SECRET (перепутали v1 API Key и v2 Secret)
    - Timestamp не в миллисекундах (13 цифр), а в секундах (10 цифр)

    Подробности — [Аутентификация v2](/api-reference/v2/authentication).
  </Accordion>
</AccordionGroup>

## Вебхуки

<AccordionGroup>
  <Accordion title="Можно ли создавать вебхуки через API?">
    **Нет.** Только через [веб-кабинет](https://loveandpay.io/dashboard/webhooks). Это политика безопасности — чтобы скомпрометированный API-ключ не позволил атакующему перенаправить ваши финансовые уведомления на свой сервер.

    Подробности — [Руководство по вебхукам](/guides/webhooks).
  </Accordion>

  <Accordion title="Сколько раз будет повторяться доставка если мой сервер не отвечает?">
    До **6 попыток** с экспоненциальным интервалом: сразу → +1 мин → +5 мин → +15 мин → +1 ч → +4 ч. После 6-й неуспешной — статус `failed`. Из «Истории доставки» можно перепослать вручную.
  </Accordion>

  <Accordion title="Какие события доступны?">
    `invoice.created`, `invoice.updated`, `invoice.paid`, `invoice.expired`, `invoice.cancelled`, `transaction.created`, `transaction.completed`, `transaction.failed`, `payment.payer_info`, `payment.third_party_payer`, `kyc.started`, `kyc.approved`, `kyc.rejected`, `kyc.expired`. Полный список с примерами payload — [Вебхуки](/guides/webhooks).
  </Accordion>

  <Accordion title="Как проверить что webhook действительно от Love&Pay?">
    Проверяйте HMAC-подпись из заголовка `X-Webhook-Signature`. Алгоритм: `HMAC-SHA256(webhook_secret, raw_body)` (подпись от сырого тела, hex). **Никогда** не доверяйте webhook без проверки подписи. Подробнее — [Вебхуки](/guides/webhooks).
  </Accordion>
</AccordionGroup>

## Возвраты

<AccordionGroup>
  <Accordion title="Как сделать возврат клиенту?">
    Через [API](/api-reference/v2/refunds/create) (`POST /api/v2/invoices/{id}/refund`) или в кабинете — [Возвраты → Создать заявку](https://loveandpay.io/dashboard/refunds). Доступно только для счетов в статусе `PAID`. Можно полный или частичный. Заявку подтверждает администратор, результат приходит вебхуком `refund.*`. Подробнее — [Возвраты](/guides/refunds).
  </Accordion>

  <Accordion title="Сколько идёт возврат?">
    1–5 рабочих дней, зависит от банка клиента. После одобрения деньги поступают на ту карту/счёт с которой была оплата.
  </Accordion>

  <Accordion title="Удерживается ли комиссия с возврата?">
    Да, согласно вашему тарифу (поле `refundFeePercent`). Удержание видно в колонке «Удержание платформы» на странице возвратов.
  </Accordion>

  <Accordion title="Можно ли несколько раз вернуть часть суммы?">
    Да. Несколько частичных возвратов на один счёт допустимы пока их сумма ≤ оплаченной.
  </Accordion>
</AccordionGroup>

## Команда и роли

<AccordionGroup>
  <Accordion title="Какие роли бывают?">
    - **PARTNER (главный)** — владелец кабинета, всё видит и управляет
    - **EMPLOYEE** (сотрудник) — видит все счета партнёра, создаёт, без управления тарифом
    - **AGENT** (кассир) — видит ТОЛЬКО свои счета (что сам создал)
    - **PARTNER (подпартнёр)** — отдельный партнёр в вашей иерархии со своими сотрудниками и комиссиями

    Подробности — [Сотрудники](/guides/employees).
  </Accordion>

  <Accordion title="Как пригласить подпартнёра?">
    [Сотрудники → Реферальная ссылка](https://loveandpay.io/dashboard/employees). Отправьте её — человек зарегистрируется как ваш подпартнёр.
  </Accordion>

  <Accordion title="Кассир видит чужие счета?">
    Нет. Роль AGENT видит только счета, созданные им самим (через POS или API под своим аккаунтом).
  </Accordion>
</AccordionGroup>

## KYC

<AccordionGroup>
  <Accordion title="Когда нужен KYC?">
    Если в настройках партнёра включён `kycEnabled` или при создании счёта передан `kycRequired: true`. Клиент пройдёт верификацию через Veriff/Didit перед оплатой.
  </Accordion>

  <Accordion title="Сколько стоит KYC?">
    По тарифной модели — пробный (10 бесплатных проверок) или платный (от $120/месяц). Тарифы в разделе [KYC](https://loveandpay.io/dashboard/kyc).
  </Accordion>

  <Accordion title="Что если KYC отклонён?">
    Сработает webhook `kyc.rejected`. Клиент увидит сообщение об отказе. Заявку можно перепроверить вручную из карточки KYC.
  </Accordion>
</AccordionGroup>

## Платформа

<AccordionGroup>
  <Accordion title="Сколько стоит использование платформы?">

    Дополнительно: KYC по тарифу, антифрод бесплатно для всех.
  </Accordion>

  <Accordion title="Где смотреть аналитику?">
    [Аналитика](https://loveandpay.io/dashboard/analytics) — конверсия, оборот, средний чек, тренды, ТОП-клиенты.
  </Accordion>

  <Accordion title="Можно ли подключить телеграм-бота для оплаты?">
    Да, есть встроенная интеграция [Telegram-боты](https://loveandpay.io/dashboard/integrations/telegram-bots).
  </Accordion>
</AccordionGroup>
