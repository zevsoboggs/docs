<Warning>
Для защиты от утечки и перепродажи ключей API работает **только** при выполнении двух условий:
1. подтверждён хотя бы один **домен** (DNS TXT);
2. запрос приходит с **задекларированного IP** вашего сервера.

Пока проверка не пройдена, запросы к API возвращают `403`. Для новых партнёров проверка
обязательна сразу; действующим партнёрам даётся срок на настройку.
</Warning>

## Зачем это нужно

Ключ (`x-api-key` + секрет) может утечь или быть скопирован. Даже с валидной подписью
чужой сервер не сможет создавать счета от вашего имени: система принимает запросы
**только с ваших доменов и IP**. Это отсекает мошеннические схемы, когда под вашим ключом
прогоняют чужой трафик.

Настройка — в панели: **Настройки → Доступ к API**.

## Шаг 1. Подтвердите домен (DNS TXT)

<Steps>
  <Step title="Добавьте домен">
    В разделе **Настройки → Доступ к API** введите домен, с которого работаете
    (например, `shop.example.com`), и нажмите «Добавить». Система выдаст токен.
  </Step>
  <Step title="Создайте TXT-запись">
    У вашего DNS-провайдера добавьте запись:

    | Поле | Значение |
    |------|----------|
    | **Тип** | `TXT` |
    | **Хост / Name** | `_loveandpay.<ваш-домен>` |
    | **Значение / Value** | `lnp-verify=<токен-из-панели>` |

    <Note>
    Если провайдер не позволяет создать поддомен `_loveandpay`, пропишите ту же TXT-запись
    на **корень** домена (`<ваш-домен>` / `@`). Оба варианта принимаются.
    </Note>
  </Step>
  <Step title="Нажмите «Проверить»">
    Вернитесь в панель и нажмите «Проверить». При успехе статус домена станет
    **«Подтверждён»** ✅.

    <Tip>
    DNS обновляется не мгновенно — обычно от нескольких секунд до нескольких минут.
    Если сразу не прошло — подождите и нажмите «Проверить» ещё раз.
    </Tip>
  </Step>
</Steps>

## Шаг 2. Задекларируйте IP сервера

Укажите IP-адреса серверов, с которых ваш бэкенд обращается к API.

<Steps>
  <Step title="Определите ваш IP">
    Это внешний (egress) IP сервера, откуда идут запросы. В панели есть подсказка
    «последний запрос к API пришёл с …» — можно добавить этот IP одной кнопкой.
  </Step>
  <Step title="Добавьте адрес(а)">
    Поддерживаются:
    - одиночный IPv4/IPv6 — `203.0.113.10`;
    - подсеть (CIDR) — `203.0.113.0/24`.

    Можно добавить несколько адресов (несколько серверов, балансировщик и т.д.)
    и указать метку для удобства.
  </Step>
</Steps>

<Info>
Указывайте **реальный IP вашего сервера**, а не адрес Cloudflare/прокси. Система определяет
исходный IP запроса (`CF-Connecting-IP` / первый в `X-Forwarded-For`).
</Info>

## Что вернётся, если проверка не пройдена

<CodeGroup>
```json Домен не подтверждён
{
  "success": false,
  "error": {
    "code": "DOMAIN_NOT_VERIFIED",
    "message": "API access requires a verified domain. Confirm your domain via DNS TXT in the dashboard."
  }
}
```

```json IP не в списке
{
  "success": false,
  "error": {
    "code": "SOURCE_IP_NOT_ALLOWED",
    "message": "Request source IP (203.0.113.99) is not in the allowlist. Add it in the dashboard."
  }
}
```
</CodeGroup>

| Код | HTTP | Что делать |
|-----|------|------------|
| `DOMAIN_NOT_VERIFIED` | 403 | Подтвердите домен (Шаг 1) |
| `SOURCE_IP_NOT_ALLOWED` | 403 | Добавьте IP сервера в список (Шаг 2) |

## Устранение неполадок

<AccordionGroup>
  <Accordion title="TXT-запись не находится">
    - Убедитесь, что хост именно `_loveandpay.<домен>` (или корень домена), без опечаток.
    - Значение — целиком `lnp-verify=<токен>`, без кавычек и лишних пробелов.
    - Дайте DNS время распространиться (до нескольких минут), повторите «Проверить».
    - Проверьте, что запись видна снаружи: `dig TXT _loveandpay.<домен>` или онлайн-чекер.
  </Accordion>
  <Accordion title="Запрос отклоняется по IP, хотя IP добавлен">
    - Сравните IP из ошибки (`Request source IP (...)`) с тем, что добавили — часто сервер
      ходит с другого адреса (NAT, несколько нод, сменился провайдер).
    - Проще всего: добавьте IP из текста ошибки, либо укажите подсеть в формате CIDR.
    - Не указывайте IP Cloudflare/прокси — нужен исходный IP вашего бэкенда.
  </Accordion>
  <Accordion title="Работало, потом перестало">
    - Сменился egress-IP сервера (переезд, автоскейл) — добавьте новый.
    - Домен перестал резолвить TXT (запись удалили) — верните её и переподтвердите.
  </Accordion>
</AccordionGroup>

## Частые вопросы

<AccordionGroup>
  <Accordion title="Сколько доменов и IP можно добавить?">
    Столько, сколько нужно. Достаточно **одного подтверждённого** домена; IP-адресов можно
    указать несколько (или подсети).
  </Accordion>
  <Accordion title="Нужно ли это, если я только тестирую?">
    Если для вашего аккаунта включена обязательная проверка — да, иначе API вернёт 403.
    Настройте домен и IP один раз, дальше всё работает.
  </Accordion>
  <Accordion title="Подтверждённый домен куда-то ещё используется?">
    Да — он автоматически подставляется как сайт мерчанта в анти-фрод-данные платёжного
    провайдера. Указывайте реальный домен, с которого принимаете оплату.
  </Accordion>
</AccordionGroup>

<Note>
Доступ к отдельным разделам (например, **Криптоплатежи** и **Криптообмены**) выдаётся
индивидуально. Если раздел вам нужен, но его нет в меню — обратитесь в поддержку:
после выдачи доступа пункт появится в панели автоматически.
</Note>
