<Info>
  **Требования**:
  - Node.js версии 19 или выше
  - Репозиторий документации с файлом `docs.json`
</Info>

Следуйте этим шагам для установки и запуска Mintlify на вашей операционной системе.

<Steps>
<Step title="Установите Mintlify CLI">

```bash
npm i -g mint
```
</Step>

<Step title="Локальный предпросмотр">

Перейдите в папку с документацией, где находится файл `docs.json`, и выполните команду:

```bash
mint dev
```

Локальный предпросмотр документации будет доступен по адресу `http://localhost:3000`.

</Step>
</Steps>

## Пользовательские порты

По умолчанию Mintlify использует порт 3000. Вы можете изменить порт с помощью флага `--port`. Например, чтобы запустить Mintlify на порту 3333, используйте команду:

```bash
mint dev --port 3333
```

Если вы попытаетесь запустить Mintlify на занятом порту, он автоматически использует следующий доступный:

```md
Port 3000 is already in use. Trying 3001 instead.
```

## Версии Mintlify

Обратите внимание, что каждый релиз CLI связан с определённой версией Mintlify. Если локальный предпросмотр не соответствует продакшн версии, обновите CLI:

```bash
npm mint update
```

## Проверка ссылок

CLI может помочь с проверкой ссылок в документации. Чтобы найти битые ссылки, используйте команду:

```bash
mint broken-links
```

## Устранение неполадок

<AccordionGroup>
  <Accordion title='Ошибка: Could not load the "sharp" module using the darwin-arm64 runtime'>

    Это может быть связано с устаревшей версией node. Попробуйте следующее:
    1. Удалите текущую версию CLI: `npm remove -g mint`
    2. Обновитесь до Node v19 или выше.
    3. Переустановите CLI: `npm i -g mint`
  </Accordion>

  <Accordion title="Проблема: Неизвестная ошибка">

    Решение: Перейдите в корень вашего устройства и удалите папку `~/.mintlify`. Затем снова запустите `mint dev`.
  </Accordion>
</AccordionGroup>

Хотите узнать, что изменилось в последней версии CLI? Смотрите [журнал изменений CLI](https://www.npmjs.com/package/mintlify?activeTab=versions).
